# BanG Dream! Live 信息查询 SOP

## When NOT to use

- 非 BanG Dream 企划相关
- 企划信息、专辑、官方消息等非 live 信息查询

## 查询流程

查询起点：

- live / event页面(WordPress站点): https://bang-dream.com/events

> **抓取优先级与 UA 门禁（决策层红线）**：
> - **优先用专用网页抓取工具**（webreader / Tavily 等：自带浏览器 UA、返回干净 markdown、省 token）；仅当环境没有这类工具，才降级到 `curl` / `py` 脚本。
> - **红线**：bang-dream.com 对 `/events/` 及所有详情页做 UA 门禁——裸 `curl`（默认 UA）一律返回 **404 空桩页（≈228B）**，易被误判成"JS 未渲染"。**任何抓取（含 curl/py 兜底）必须带浏览器 UA。**
> - 工具判定标准、示例名单、curl/py 骨架见 [网页抓取策略](#网页抓取策略)。

### 1. 从 live/event 页面定位演出

访问 live / event 页面，根据用户描述，通过页面提供的筛选功能筛选目标演出,调用参考 [live/event页面筛选说明](#live/event页面筛选说明)。

页面有分页逻辑，注意是否完全获取到。

页面中每个信息卡片包含：
- 类型标签：Live / Event
- 演出名称
- 开催日时（可能未公布）
- 场所（可能未公布）

信息卡片对应 live & event 的其他详细信息需要跳转到对应的超链接(详情页)中获取。

### 2. 详情页处理多场次情况

获取到的 live / event 列表中，一个巡演/live只有**一个入口**。

进入详情页后，注意页面内是否有**场次切换导航**（如"大阪公演""東京公演"等）。不同场次的 URL 不同，需要根据描述导航切换到对应的页面采集。

### 3. 识别详情页信息完整度

通过以下方式判断页面信息是否已完整更新：

**信息不完整的特征**：
- 特典内容写"後日公開"
- 部分模块内容缺失或标注"調整中"
- 販売情報只有"近日公開"等占位文字

**信息完整的特征**：
- 各模块有具体内容
- 販売情報有明确的受付期间和URL
- 特典、商品等有详细说明或图片

### 4. 通过导航栏采集信息

详情页顶部导航栏列出该演出当前已公布的所有模块。逐模块采集，如果模块内有其他url，也需要获取对应url的内容作为该模块的信息：

#### 公演概要
- 公演名称
- 日程：区分多日公演的 DAY1/DAY2 等，包含開場/開演时间
- 可能有 Opening Act 信息
- 会場
- 出演

#### 会場チケット
- 料金：区分席位类型（プレミアムシート/一般指定席/見切れ席等）
- 是否含特典（特製グッズ付き）
- 特典内容：通常以图片展示，需用视觉模型识别
- 座席イメージ：图片，需用视觉模型识别座位布局
- 販売情報：可能有多轮先行抽选/一般发售，注意受付状态

#### チケットトレード（如有）
- 官方二手票务交易平台链接
- 受付期間

#### 配信チケット（如有）
- 线上直播料金
- 販売期間 / 配信期間
- 购入链接（区分国内/海外）

#### グッズ通販（如有）
- 通販链接（通常指向 bushiroad-store.com）
- 根据语境判断是否需访问该链接获取商品详情

#### グッズ販売（如有）
- 販売商品：通常以图片展示，需用视觉模型识别
- 販売場所 / 販売日時
- 購入制限
- 整理券制度：可能有ブシロードカード会员优先、随机整理券、最后尾整理券等
- 支付方式

#### 联动ブース（如有）
- 联动品牌及商品信息
- 限定特典

#### CD·Blu-ray販売（如有）
- 販売商品图片：需用视觉模型识别
- 購入特典：按购买金额赠送的特典卡等
- 抽選会：景品内容、参加条件

#### カプセルトイ（如有）
- 扭蛋商品、价格、场所

#### 注意事項
- 可能内嵌在详情页，也可能链接到外部通用规则页
- 主要为禁止行为列表和入场规则

#### お祝い花（如有）
- 花篮尺寸限制
- 到着指定日時 / 回収指定日時
- NG示例图片：需用视觉模型识别
- 送付先地址

#### お問合せ
- 通常分为「公演に関するお問い合わせ」和「チケットに関するお問い合わせ」两个渠道

## 需要视觉模型处理的内容

以下内容通常以图片形式展示，模型支持视觉理解时才做解答，否则提供给用户url让其自行查看：

| 内容 | 分析要点 |
|------|---------|
| 特典图片 | 种类、样式、获取条件 |
| 座席イメージ | 区域划分、席位类型分布、价格标注 |
| 周边商品图 | 商品种类、价格、限定信息 |
| CD/BD商品图 | 商品名、版本、价格、特典 |
| 花篮NG示例 | 不允许的花篮样式特征 |


## Output

只输出获取到的已确认信息，包括详情页内url中所包含的信息。不输出信息完整度判断的内容。
